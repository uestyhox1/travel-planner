## 给普通用户：直接运行

### 你只需要做两件事

```
1. 下载 dist/ 文件夹到你的电脑（任意位置）
2. 双击 TravelPlanner.exe
```

就这么简单。程序会自动打开一个桌面窗口。

> **需要什么电脑？** Windows 10 或 11 即可。不需要 Python，不需要安装任何东西。

### 第一次使用

1. 窗口打开后，点击「注册」创建一个账号
2. 用户名 2-20 个字符，密码至少 4 位
3. 输入图片验证码（4 个字符）
4. 注册成功后自动登录，开始使用

> 也内置了一个默认管理员账号：`admin` / `admin`

### OCR 识别（图片转文字）

**直接就能用！** 不需要额外安装任何东西。便携版 Tesseract OCR 已经打包在 `dist/tesseract/` 文件夹里了。

支持的语言：中文简体 + 英文混合识别。

## 给开发者：从源码运行

### 1. 克隆项目

```bash
git clone https://github.com/uestyhox1/travel-planner.git
cd travel-planner
```

### 2. 安装依赖 & OCR（一键搞定）

```bash
# 安装 Python 依赖（需要 Python 3.10+）
pip install -r requirements.txt

# 一键安装 OCR 引擎（自动检测、下载、配置）
python setup_ocr.py
```

`setup_ocr.py` 会自动：
1. 检查是否已有便携版 Tesseract
2. 查找系统中已安装的 Tesseract
3. 都没找到就通过 `winget` 自动安装
4. 下载中英文语言包（eng + chi_sim）

> 不想用 OCR？跳过 `setup_ocr.py` 即可，手动输入文字不受影响。

### 3. 启动

```bash
# 桌面窗口模式（默认）
python app.py

# 浏览器模式（方便调试）
python app.py --browser
# 然后访问 http://127.0.0.1:5000
```

## 功能详解

### 📷 上传攻略 & OCR 识别

这是整个工具的核心入口。你有一张旅行攻略的截图或照片，想把里面的文字提取出来，自动生成行程。

**操作步骤：**

1. 左侧导航点击「**上传攻略**」
2. 拖拽图片到上传区域，或点击选择文件
3. 系统自动进行 OCR 文字识别
4. 识别出的文字自动解析为结构化行程
5. 切换到「**行程总览**」即可看到按天整理好的时间线

**支持的手动输入方式：**

如果图片质量不好或没有图片，也可以直接粘贴文字：

1. 在「上传攻略」页面切换到「**文本输入**」标签
2. 粘贴攻略文字（支持多种格式）
3. 点击「解析」，系统自动提取日期和活动

**支持的文本格式：**

```
第1天：北京
上午：故宫（提前预约）
中午：附近吃饭
下午：天坛公园

Day 2：长城
08:00 出发去八达岭长城
12:00 午餐
15:00 返回市区
```

系统会自动识别「第X天」「Day X」「X号」「上午/下午/晚上」「HH:MM 时间格式」等多种写法。

### 📅 管理行程

识别完成后，在「**行程总览**」页面查看和管理。

- **顶部标签页**：按天切换，每天一个标签
- **时间线视图**：按时间顺序排列所有活动
- **勾选完成**：点击活动左侧圆圈标记完成/未完成
- **添加活动**：点「+ 添加活动」手动增加
- **编辑活动**：点击活动文字即可编辑内容、时间、地点、备注
- **删除活动**：点击右侧 ✕ 按钮

**活动分类：** 景点 / 餐饮 / 住宿 / 交通 / 购物（用不同颜色标识）

**顶部统计栏** 显示：总天数 | 总活动数 | 完成进度百分比

### 🏷️ 景点管理 & 小红书攻略

「**景点管理**」页面自动汇总行程中出现的所有地点。

**AI 自动提取（需配置 API Key）：**
1. 点击「AI 提取景点」按钮
2. AI 自动从 OCR 文本中识别景点名称
3. 为每个景点生成一句话简介
4. 自动关联小红书攻略帖子

**手动搜索景点：**
1. 在搜索框输入景点名称
2. 点击「搜索」，系统返回相关攻略
3. 点击「保存」加入景点列表

**景点卡片颜色含义：**
- 🟣 紫色边框 = AI 自动识别的景点
- 🟢 绿色边框 = 手动搜索添加的景点
- ⚪ 灰色边框 = 手动创建的景点

**点击景点卡片** → 展开详情，查看小红书攻略帖子 → 点击帖子可跳转到小红书网页。

**打开小红书面板：** 点击右上角 📕 图标，可以在面板中直接搜索、浏览小红书攻略。

### ✅ 待办清单

「**待办清单**」页面管理旅行前的准备工作。

- **添加待办**：输入内容，设置优先级（普通/重要/紧急）和截止日期
- **勾选完成**：点击待办左侧圆圈
- **筛选**：全部 / 未完成 / 已完成 / 高优先级
- **进度条**：顶部显示完成百分比

**常见待办示例：**
- 订机票/火车票
- 预订酒店
- 预约景点门票（故宫、长城等需要提前预约）
- 购买旅行保险
- 换汇/准备现金
- 下载离线地图

### 🤖 AI 助手

「**AI 助手**」页面可以跟 AI 对话，获取旅行建议。

**配置 API Key（必须）：**

1. 点击右上角 ⚙️ 设置
2. 切换到「**API 配置**」标签
3. 填入 API Key：

| 服务商 | 获取地址 | 默认模型 |
|--------|---------|---------|
| DeepSeek（推荐，便宜） | [platform.deepseek.com](https://platform.deepseek.com) | `deepseek-chat` |
| Anthropic Claude | [console.anthropic.com](https://console.anthropic.com) | `claude-sonnet-4-6` |
| OpenAI 兼容接口 | 各服务商 | 自定义 |

4. 点击「保存」

**快捷提问按钮：**
- 「行程合理吗？」—— AI 分析你的行程安排
- 「推荐美食」—— 根据目的地推荐餐厅
- 「穿搭建议」—— 根据季节和目的地给出建议
- 「交通攻略」—— 分析交通方案

AI 会读取你当前的行程数据，给出**针对性的个性化建议**，而不是泛泛而谈。

**没有 API Key 也能用：** 系统会生成格式化提示词，复制到 Claude Code 或其他 AI 工具中即可。

### 📄 导出 PDF

1. 在任意页面点击顶部「**导出 PDF**」按钮
2. 系统自动生成包含以下内容的 PDF 文档：
   - 封面（标题 + 日期 + 统计概要）
   - 每日行程表（时间、活动、地点、备注）
   - 待办清单
   - 景点参考信息

> PDF 使用中文字体，排版整洁，可以直接打印。

## 常见问题

### Q: OCR 识别不准怎么办？

1. **图片质量要好**：尽量使用清晰的截图，避免照片模糊、反光、倾斜
2. **手动修正**：在「上传攻略」页面切换到「文本输入」，粘贴并手动修改后再解析
3. **逐条编辑**：识别完成后，在行程总览中逐条编辑修正

### Q: 程序打开了但是窗口是空白的？

- 尝试用浏览器模式：`python app.py --browser`
- 检查防火墙是否拦截了本地 5000 端口
- Windows 7 可能不兼容（需要 Windows 10+）

### Q: 数据存在哪里？

- 所有数据存在 `data/trips.db`（SQLite 数据库）
- 上传的图片存在 `data/uploads/`
- API 配置存在 `data/config.json`
- 换电脑时复制整个 `data/` 文件夹即可迁移数据

### Q: 忘记密码怎么办？

- 当前版本：删除 `data/trips.db` 重置所有数据（会丢失已有行程）
- 使用默认账号 `admin / admin` 重新登录

### Q: 怎么换到另一台电脑？

```
1. 把整个 dist/ 文件夹复制到新电脑
2. 把 data/ 文件夹也复制过去（放在 TravelPlanner.exe 同级目录）
3. 双击运行 —— 所有数据都在
```

### Q: OCR 支持繁体中文吗？

当前内置的语言包是简体中文（chi_sim）。如需繁体中文，下载 `chi_tra.traineddata` 放入 `tesseract/tessdata/` 即可。

### Q: 可以多人同时使用吗？

当前版本是单机桌面应用，不支持多人协作。每个人在自己的电脑上独立使用。

## 构建 & 部署

### 构建 exe

```bash
# 一键构建
build_exe.bat

# 输出在 dist/ 文件夹
#   - TravelPlanner.exe  (20MB)
#   - tesseract/         (207MB, OCR引擎)
```

### 部署到别人电脑

```
把整个 dist/ 文件夹发给对方 → 双击 TravelPlanner.exe → 开始使用
```

对方不需要装 Python，不需要装 Tesseract，什么都不需要。

### 项目结构速览

```
旅行攻略/

├── parser.py              # 文本解析器

├── build_exe.bat          # 一键打包脚本
├── requirements.txt       # Python 依赖
```
